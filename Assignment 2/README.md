## Задача 1. Введение в гетерогенную параллелизацию

### Что такое гетерогенная параллелизация
Гетерогенная параллелизация - это подход, при котором для решения одной задачи используются разные вычислительные устройства, чаще всего CPU и GPU. Каждое устройство выполняет ту часть работы, для которой оно наиболее эффективно.

### Различия между параллельными вычислениями на CPU и GPU
CPU содержит небольшое количество универсальных и мощных ядер, которые хорошо подходят для сложных логических и последовательных операций. GPU имеет большое количество более простых ядер и ориентирован на массовую параллельную обработку однотипных данных.

### Преимущества гетерогенной параллелизации
Гетерогенная параллелизация позволяет повысить производительность, сократить время выполнения программ и более эффективно использовать аппаратные ресурсы, распределяя задачи между CPU и GPU.

### Примеры реальных приложений
Гетерогенная параллелизация используется в машинном обучении, обработке изображений и видео, научных расчетах, моделировании физических процессов и компьютерной графике.

## Контрольные вопросы к Assignment 2  
(OpenMP, CUDA и гетерогенные вычисления)

### 1. Что понимается под гетерогенной параллелизацией?
Под гетерогенной параллелизацией понимается использование разных типов вычислительных устройств, таких как CPU и GPU, для совместного выполнения вычислений в одной программе.

### 2. В чем принципиальные различия архитектур CPU и GPU?
CPU ориентирован на выполнение сложных последовательных задач и имеет небольшое количество мощных ядер, тогда как GPU состоит из большого числа простых ядер и предназначен для параллельной обработки больших объемов данных.

### 3. Какие типы задач лучше подходят для выполнения на GPU, а какие — на CPU?
GPU лучше подходит для задач с большим количеством однотипных операций, например обработки массивов, матриц и изображений. CPU более эффективен для задач с ветвлениями, сложной логикой и последовательными вычислениями.

### 4. Почему не все алгоритмы эффективно распараллеливаются с использованием OpenMP?
Не все алгоритмы можно эффективно распараллелить, потому что между их шагами могут существовать зависимости по данным, а также из-за накладных расходов на создание и синхронизацию потоков.

### 5. В чем заключается основная идея алгоритма сортировки слиянием?
Алгоритм сортировки слиянием основан на принципе «разделяй и властвуй»: массив разбивается на подмассивы, которые сортируются отдельно, после чего происходит их слияние в один отсортированный массив.

### 6. Какие сложности возникают при реализации сортировки слиянием на GPU?
Основные сложности связаны с организацией параллельного слияния, синхронизацией потоков, управлением памятью GPU и выбором эффективной схемы разбиения данных.

### 7. Как выбор размера блока и сетки влияет на производительность вычислений на GPU?
Размер блока и сетки влияет на загрузку вычислительных ресурсов GPU. Неправильный выбор может привести к неэффективному использованию ядер и снижению производительности.

### 8. Почему гетерогенный подход может быть эффективнее использования только CPU или только GPU?
Гетерогенный подход позволяет использовать сильные стороны и CPU, и GPU, распределяя задачи оптимальным образом, что часто дает более высокую производительность по сравнению с использованием только одного типа устройства.
